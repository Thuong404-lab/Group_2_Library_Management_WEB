Bạn đang làm việc trong repository Library Management System dùng Spring Boot, Thymeleaf và MessageSource.

Thông tin tích hợp:
- Nhánh hiện tại của tôi chứa chức năng tôi đang làm dở và phải được giữ nguyên: <NHÁNH_HIỆN_TẠI>
- Nhánh hoặc commit chứa chức năng song ngữ Anh–Việt cần tích hợp: <NHÁNH_SONG_NGỮ>
- Không được push lên remote nếu tôi chưa yêu cầu.

MỤC TIÊU

Hãy merge chức năng song ngữ từ <NHÁNH_SONG_NGỮ> vào nhánh hiện tại, đồng thời giữ nguyên toàn bộ chức năng tôi đang làm dở.

Kết quả cuối cùng phải có cả:
1. Chức năng của tôi.
2. Chức năng song ngữ Anh–Việt.
3. Không mất code của bất kỳ bên nào.
4. Không làm thay đổi endpoint, form action, API, database mapping hoặc nghiệp vụ ngoài phạm vi cần thiết.
5. Những giao diện và backend mới của tôi cũng phải được bổ sung song ngữ đầy đủ.

CÁCH MERGE

1. Đọc và hiểu thay đổi của cả hai nhánh trước khi giải quyết conflict.
2. Thực hiện merge theo hướng có thể review, ưu tiên `git merge --no-commit`.
3. Với từng conflict:
   - Đọc base, ours và theirs.
   - Kết hợp theo ý nghĩa nghiệp vụ.
   - Không chỉ xóa marker conflict rồi chọn đại một bên.
4. Sau mỗi file được xử lý, kiểm tra lại:
   - import
   - constructor injection
   - annotation
   - endpoint
   - form action
   - tên model attribute
   - tên request parameter
   - JavaScript selector
   - ID HTML
   - DTO và repository method signature
5. Tìm và loại bỏ toàn bộ marker:
   - `<<<<<<<`
   - `=======`
   - `>>>>>>>`

QUY TẮC GIẢI QUYẾT CONFLICT I18N

1. Conflict trong `messages.properties` và `messages_vi.properties`

- Phải giữ hợp nhất tất cả key của cả hai bên.
- Không xóa key đang được template/backend sử dụng.
- Mỗi key English phải có key tương ứng trong file Vietnamese.
- Không tạo key trùng.
- Không đổi ý nghĩa key cũ nếu đang được nhiều nơi sử dụng.
- Key mới nên đặt theo module, ví dụ:
  - `member.featureName.*`
  - `librarian.featureName.*`
  - `admin.featureName.*`
  - `backend.featureName.*`
  - `validation.featureName.*`
  - `notification.featureName.*`

2. Conflict trong template Thymeleaf

Phải giữ nguyên:

- `th:action`
- `th:href`
- `th:object`
- `th:field`
- `th:value`
- `name`
- `id`
- `data-*`
- Bootstrap modal target
- JavaScript selector
- điều kiện `th:if`, `th:unless`
- vòng lặp `th:each`

Các chuỗi giao diện viết cứng phải chuyển sang message key:

- Nội dung:
  `<span th:text="#{feature.key}">English fallback</span>`

- Placeholder:
  `th:placeholder="#{feature.placeholder}"`

- Title:
  `th:title="#{feature.title}"`

- Aria label:
  `th:aria-label="#{feature.ariaLabel}"`

- Message có tham số:
  `th:text="#{feature.key(${value})}"`

Fallback HTML nên dùng English vì English là mặc định.

Không được dịch giá trị gửi về backend nếu đó là giá trị nghiệp vụ. Ví dụ:

```html
<option value="Mới"
        th:text="#{librarian.inventory.condition.new}">
    New
</option>
value="Mới" được giữ nguyên nếu backend/database đang dùng giá trị đó; chỉ phần hiển thị được dịch.
Conflict trong JavaScript
Không viết cứng thông báo English hoặc Vietnamese trong JavaScript.
Với script inline của Thymeleaf, dùng:
const featureI18n = {
    required: /*[[#{validation.required}]]*/ "This field is required.",
    confirm: /*[[#{common.confirmQuestion}]]*/ "Are you sure?"
};
Hoặc truyền thông báo qua data-*:
<button th:data-confirm-message="#{feature.confirmMessage}">
Sau đó JavaScript đọc element.dataset.confirmMessage.

lỗi API hiển thị trên trang
